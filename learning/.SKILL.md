# Code generating criterion

## 1. 子项目目标

`/learning` 子项目用于实现和测试 continuous attractor（连续吸引子）相关的 local learning rule（局部学习规则）模型。当前优先目标是 Vafidis-style predictive local plasticity toy model，用于验证局部预测型可塑性能否学习出：

1. HD bump maintenance（头朝向活动包维持）；
2. velocity-driven path integration（角速度驱动路径积分）；
3. 具有明确物理含义的 recurrent weight structure（递归权重结构）与 HR-to-HD asymmetric weight structure（头旋转到头朝向的不对称权重结构）。

该子项目不是用于 notebook 中临时堆代码。Notebook 只用于说明、调用、可视化与记录实验结果；核心模型、动力学、学习规则、分析函数必须放在 `src/learning/` 中。

## 2. 包管理与运行方式

当前使用 `conda` 管理环境，不使用 `uv`。推荐在 `/learning` 下保留：

## 4. 配置管理原则

配置文件只负责参数，不写代码逻辑。核心配置分成五类：

1. `model/`：网络规模、权重初始化、激活函数、全局抑制；
2. `simulation/`：`dt`、训练时长、测试时长、随机种子、保存间隔；
3. `learning_rule/`：learning rate、PSP trace time constant、weight decay、clipping；
4. `stimulus/`：visual teacher、angular velocity input；
5. `paths/`：输出路径、图像路径、run directory。

`configs/experiments/*.yaml` 负责组合这些配置。当前阶段可以手动在 `vafidis_toy.yaml` 中写完整参数，不强制实现 Hydra-style config composition。

配置读取后必须保存一份 resolved config：

```text
runs/vafidis_toy/<run_id>/config_resolved.yaml
```

保证后续结果可复现。

## 5. 代码命名规则

Python 变量使用 `snake_case`。变量名必须体现物理含义，禁止使用不可解释的短变量名，例如：

```text
x, y, W1, W2, R, I, tmp, data, output
```

权重矩阵统一使用 source-to-target 命名：

```text
w_hd_to_hd
w_hr_to_hd
w_lhr_to_hd
w_rhr_to_hd
w_hd_to_hr
```

神经活动统一使用 population 名称：

```text
r_hd
r_hr
r_lhr
r_rhr
```

输入电流统一使用 input source 与 target：

```text
i_vis_to_hd
i_vel_to_hr
i_vel_to_lhr
i_vel_to_rhr
```

compartment 变量必须标明 HD 与 compartment：

```text

v_hd_ss
v_hd_proximal
```

学习规则变量必须标明其局部物理意义：

```text

p_hr

```

角度变量必须区分真实值与网络解码值：


## 6. 模块职责划分

`models/vafidis_toy.py` 只负责模型状态、参数与单步更新，不负责画图和文件保存。

`stimuli/` 只负责生成外部输入，例如 visual bump 和 angular velocity input。

`dynamics/` 只负责神经动力学，例如 HD/HR current 更新、activation function。

`plasticity/` 只负责局部学习规则与 PSP traces，不允许引入全局 loss、backpropagation 或 autograd。

`analysis/` 只负责从保存结果中计算指标，例如 PI error、velocity gain、bump drift。

`plotting/` 只负责画图，不改变模型状态。

`experiments/` 是命令入口，负责读取 config、初始化模型、运行训练/测试、保存结果。

`notebooks/` 只允许调用 `src/learning` 中的函数，不允许在 notebook 中重新实现核心模型。

## 7. Vafidis toy model 的核心数据流

每个 timestep 必须按以下顺序执行：

1. 更新真实变量 `theta_true` 和 `angular_velocity`；
2. 根据 `theta_true` 生成 `i_vis_to_hd`；
3. 根据 `angular_velocity` 生成 `i_vel_to_hr`；
4. 更新 HR 的低通 HD 输入 `r_hd_to_hr_lp`，并把即时速度输入加到 HR voltage/current `i_hr`；
5. 计算 HR firing rate `r_hr`；
6. 更新 HD distal input current `i_hd_distal`；
7. 按 `tau_l_hd dVd/dt = -Vd + Id` 更新 `v_hd_distal`；
8. 计算 `v_hd_ss = p_distal_to_proximal * v_hd_distal`；
9. 计算 `v_hd_proximal = v_hd_ss + i_vis_to_hd`；
10. 计算 HD firing rate `r_hd = activation_sigmoid(v_hd_proximal)`；
11. 计算 learning error `e_hd = activation_sigmoid(v_hd_proximal) - activation_sigmoid(v_hd_ss)`；
12. 按 `tau_s` 与 `tau_l_hd` 的两级滤波更新 PSP traces `p_hd` 和 `p_hr`；
13. 若处于 training phase，则先更新 plasticity-induction traces `delta_w_*`，再更新 `w_hd_to_hd` 与 `w_hr_to_hd`；
14. 执行数值稳定处理，例如去除自连接、clipping、NaN / Inf 检查；
15. 保存必要的 history。

testing phase 必须冻结权重。除 bump initialization 外，不允许在 darkness test 中使用 visual teacher。

## 9. 测试与 sanity checks

必须实现基础 shape check：

```python
assert r_hd.shape == (n_theta,)
assert r_hr.shape == (n_hr,)
assert w_hd_to_hd.shape == (n_theta, n_theta)
assert w_hr_to_hd.shape == (n_theta, n_hr)
assert w_hd_to_hr.shape == (n_hr, n_theta)
```

参考 Vafidis 主模型时，`n_theta = n_hr = 60`，其中 HD 细胞为 odd/even 成对的 30 个角位置，HR 总数为 60，左 / 右 HR wing 各 30。当前 toy 使用 `[-π, π)` 角度区间；按 release code 的 0-based 索引，even HD 投到第一半 HR，odd HD 投到第二半 HR。

必须检查：

1. firing rate 非负；
2. 所有数组无 NaN / Inf；
3. 角度差使用 circular difference，不能直接普通相减；
4. `w_hd_to_hd` 的 diagonal 在更新后应为 0；
5. training phase 与 testing phase 的权重更新开关必须明确；
6. random seed 必须可复现。

## 10. 第一阶段成功标准

第一阶段不要求完全复现 Vafidis et al. 的论文结果。最小成功标准是：

1. 代码可以从 config 启动训练；
2. 训练后保存完整 run directory；
3. `w_hd_to_hd` 出现局部对称连接结构；
4. `w_lhr_to_hd` 与 `w_rhr_to_hd` 出现方向相反的偏移结构；
5. bump maintenance test 中 decoded heading 不快速崩溃；
6. darkness test 中 decoded heading 能随 angular velocity 同方向移动；
7. velocity gain curve 可以输出。

若第一版无法达到 gain (g\approx 1)，不视为失败。优先保证代码结构、变量含义、可复现性和可诊断性。
