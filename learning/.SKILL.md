    各部分职责如下:

    - `.todo/TODO.md`

        记录当前阶段的具体任务。Agent 在生成或修改代码前, 必须优先阅读该文件, 明确本次任务目标, 不应擅自扩展到 TODO 之外的大型重构。

    - `configs/`

        存放实验配置文件。配置文件用于控制神经元数量、离散角度数量、Gaussian process kernel 参数、激活函数参数、学习率、正则化强度、训练 epoch、RNN 动力学仿真时间等。脚本中不应硬编码大量实验参数, 应尽量从配置文件读取。

        其中:

        - `ring_batch_baseline.yaml`: 用于复现 Clark-style batch weight construction 的最小 baseline。
        - `ring_local_learning.yaml`: 用于测试 online/local learning rule 是否能逼近 batch 解。
        - `ring_partial_input.yaml`: 用于测试 partial structured input 是否足以诱导低维 ring manifold。
        - `place_1d_synthetic.yaml`: 预留给后续 place cell / 1D spatial manifold 实验, 初期除非 TODO 指定, 不需要实现。

    - `scripts/`

        存放可直接运行的实验入口。每个脚本只负责一个明确阶段, 不应写成包含大量分支的大型总控程序。

        推荐流程为:

        1. `run_01_generate_teacher_manifold.py`

            生成 teacher manifold, 包括低维变量 `theta`, 输入电流场 `x_star(theta)`, 输出 firing-rate tuning curves `phi_star(theta)`。结果保存为 `.npz` 文件, 并输出 tuning curve 图像。

        2. `run_02_batch_baseline.py`

            使用 batch least-squares / ridge regression 构造 baseline recurrent weight matrix `J_batch`, 用作 local learning 的参照。

        3. `run_03_local_learning.py`

            使用指定的 local learning rule 训练 `J_local`, 记录 residual、weight norm、diagonal constraint violation 等训练过程指标。

        4. `run_04_simulate_learned_rnn.py`

            对学习得到的 RNN 进行动力学仿真, 检查初态扰动后是否回到目标 manifold, 以及沿 manifold 的漂移情况。

        5. `run_05_compare_batch_vs_local.py`

            比较 `J_batch` 和 `J_local` 的 weight matrix、spectrum、manifold reconstruction residual、dynamical stability 等指标。

        6. `run_smoke_test.py`

            使用很小的参数规模快速检查完整流程是否能运行。例如 `N=32`, `n_theta=64`, `n_steps=1000`。该脚本只用于检查代码正确性, 不用于产生正式科研图像。

    - `src/learning/`

        存放可复用代码。原则上, `src` 中的函数不应直接读写固定路径, 除非该函数位于 `io_utils.py`。实验路径、参数和输出位置应由 `scripts` 或 `configs` 控制。

        推荐模块职责如下:

        - `config.py`

            负责读取 YAML 配置文件, 将配置转换为 Python 字典或 dataclass。不得在多个脚本中重复写解析配置的代码。

        - `io_utils.py`

            负责创建输出文件夹、保存 `.npz`、保存 `.json`、读取中间结果等。所有文件路径相关操作优先放在此处。

        - `manifolds.py`

            负责生成低维连续变量的离散网格。例如 ring attractor 中的 `theta in [0, 2pi)`, 未来可扩展到 line attractor、1D place field、2D torus 等。

        - `kernels.py`

            负责 Gaussian process covariance kernel 的构造。第一阶段至少应包含 periodic RBF kernel:

            ```python
            Gamma_x(theta, theta') = sum_n exp[-sigma^2 / 2 * (theta - theta' + 2*pi*n)^2]
            ```

            后续若扩展到 place cell, 可加入普通 RBF kernel 或 translational invariant kernel。

        - `activations.py`

            负责神经元非线性激活函数及其导数。例如 softplus、normalized softplus、erf activation 等。若某个动力学稳定性指标需要 Jacobian, 则必须同时实现对应的 activation derivative。

        - `teacher.py`

            负责生成 teacher manifold, 即根据 Gaussian process 采样 `x_star(theta)`, 再通过 activation 得到 `phi_star(theta)`。该模块只负责生成目标 manifold, 不负责学习权重。

        - `batch_solver.py`

            负责实现 batch baseline, 例如 row-wise ridge regression 或 Clark-style least-squares solution。该模块用于提供 local learning 的比较对象, 不应混入 online learning 逻辑。

        - `local_rules.py`

            负责定义局部学习规则。例如:

            ```python
            Delta J_ij = eta * error_i(theta_t) * phi_j(theta_t) - eta * lambda * J_ij
            ```

            其中 `error_i(theta_t)` 可以是 teacher forcing 下的 postsynaptic current error。每一种 learning rule 应写成独立函数, 并在 docstring 中说明其数学形式。

        - `trainer.py`

            负责组织 local learning 的训练循环, 包括 theta 采样、调用 local update、记录 loss/residual、保存 checkpoint 等。具体学习规则应从 `local_rules.py` 调用, 不应直接写死在训练循环中。

        - `dynamics.py`

            负责 RNN 动力学仿真:

            ```python
            tau dx_i/dt = -x_i + sum_j J_ij phi_j(x_j) + b_i
            ```

            初期使用 Euler method 即可。除非 TODO 明确指定, 不应擅自引入复杂 ODE solver 或 GPU 加速。

        - `metrics.py`

            负责定量指标计算, 包括:

            - `J phi_star(theta) ≈ x_star(theta)` 的 reconstruction residual
            - 当前状态到目标 manifold 的最近距离
            - 最近 manifold index / decoded theta
            - normal convergence time
            - tangential drift speed
            - Jacobian eigenvalue spectrum
            - learned weights 和 batch weights 的差异

        - `plotting.py`

            负责标准化绘图, 包括 tuning curves、weight matrix、COM 排序后的 weight matrix、eigenvalue spectrum、training loss、manifold distance vs time、PCA trajectories 等。绘图函数应接收数据和保存路径, 不应在函数内部重新运行实验。

        - `diagnostics.py`

            负责失败诊断工具。尤其用于分析“初态无法收敛到 manifold”的情况, 例如:

            - 检查 `J phi_star - x_star` 是否足够小
            - 检查 activation inverse / normalization 是否一致
            - 检查 diagonal constraint 是否破坏训练
            - 检查 uniform inhibition 参数是否导致 mean mode 不稳定
            - 检查 Jacobian 是否存在正实部 eigenvalue
            - 检查仿真 dt 是否过大
            - 检查目标 manifold 是否经过了额外 normalization 但仿真中未同步使用

    - `notebooks/`

        仅用于交互式检查和画临时图。Notebook 不应成为主要实验入口。任何在 notebook 中确认有用的代码, 都应整理回 `src/learning/` 或 `scripts/` 中。

    - `outputs/`

        存放实验输出。该文件夹中的大文件不应提交到 git。正式输出建议按照实验名进一步分子文件夹, 例如:

        ```text
        outputs/
        └── ring_local_learning/
            ├── arrays/
            ├── figures/
            ├── logs/
            └── reports/
        ```

        其中:

        - `arrays/`: 保存 `.npz` 或 `.npy` 中间结果。
        - `figures/`: 保存 `.png` 或 `.pdf` 图像。
        - `logs/`: 保存训练过程日志。
        - `reports/`: 保存 `.json` 或 `.md` 格式的实验摘要。

    - `tests/`

        存放最小单元测试。测试不需要覆盖所有科研逻辑, 但必须覆盖最容易 silently fail 的基础函数, 例如 kernel 是否对称、activation 输出是否有限、batch solver 输出维度是否正确、RNN 仿真是否产生 NaN、manifold distance 是否返回合理形状等。

    代码组织原则:

        1. 第一阶段只实现 ring attractor / head direction 相关内容, 不要同时展开 place cell、grid cell、line attractor 等多个方向。

        2. 新增函数优先放入 `src/learning/` 中的对应模块, 不要在 `scripts/` 中堆积大量函数定义。

        3. 每个 `scripts/run_xx_*.py` 都应该可以从 `/learning` 目录直接运行。

        4. 每次实验必须保存关键中间结果, 避免重复长时间计算。

        5. 对于 local learning, 必须同时保存训练过程曲线和最终动力学验证结果。仅有 loss 下降不足以证明形成了 attractor。

        6. 若发现动力学不收敛, 不应直接大改模型; 应优先调用或补充 `diagnostics.py` 中的诊断函数, 判断失败来自 weight reconstruction、activation mismatch、normal instability、mean mode instability、还是 numerical integration 问题。

        7. 除非 TODO 明确要求, 不应引入深度学习框架。第一阶段优先使用 NumPy、SciPy、Matplotlib 完成小规模可解释实验。
